# Design Docs Overview

This folder now has two documentation layers:

- `PFE_BOOK/`: canonical, cleaned sprint material ready to reuse in the PFE report.
- legacy sprint folders (`00_...` to `10_...`): working notes, draft revisions, and archived snapshots.

## Recommended Source for Final Writing

Use the files in `designDocs/PFE_BOOK/` first, then pull details/screenshots from legacy folders when needed.

## Naming Convention

- Use `Sprint NN: <descriptive title>` for human-facing chapter headings.
- Keep `xx` only in scratch/draft paths when the sprint number is intentionally provisional.
- Prefer one canonical merged file per sprint in `PFE_BOOK/`, with drafts preserved in the source folder for traceability.
