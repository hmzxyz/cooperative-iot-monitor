# Sprint 09 Progress Report (2026-05-03)

## Summary

Implemented the technician authentication webpage flow end-to-end for the current MES build:
- Technician sign in
- Technician self sign up
- Forgot password with security question + answer

This update aligns the frontend UX with the existing backend user model and persists accounts in the DB.

## Implemented Features

### Frontend

- Reworked `frontend/src/pages/LoginPage.jsx` into 3 views:
  - `Sign In`
  - `Sign Up`
  - `Forgot Password`
- Added clear success/error feedback for account creation and password reset.
- Added view-switch controls to move between auth flows.
- Styled new auth states and navigation in `frontend/src/styles.css`.

### Frontend Auth Service

- Extended `frontend/src/context/AuthContext.jsx` with:
  - `registerTechnician(...)`
  - `getPasswordResetQuestion(username)`
  - `resetPassword(...)`

### Backend

- Improved validation and normalization in `backend/app/routers/auth.py`.
- Added `GET /api/auth/password-reset/question?username=...`.
- Kept `POST /api/auth/password-reset` and aligned it to the frontend flow.
- Added weak-password rejection and duplicate-username protections.

### Tests

- Added backend smoke tests for:
  - Technician default role registration
  - Recovery question retrieval
  - Password reset with invalid and valid security answers
  - Login with updated password
