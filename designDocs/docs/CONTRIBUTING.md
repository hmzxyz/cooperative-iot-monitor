# Contributing to Cooperative IoT Monitor

We welcome contributions to improve this project! Please follow these guidelines:

## Before Contributing
1. Fork the repository
2. Create a new branch for your changes (e.g., `feature/sprint-0-auth`)
3. Make sure your local setup matches the `getting-started.md` instructions

## Code Style
- Use consistent formatting (prettier/rustfmt if applicable)
- Follow API naming conventions in backend
- Keep React component names in PascalCase

## Bug Reports
- File new issues in GitHub with:
  - Reproduction steps
  - Expx^x^ected vs actual behavior
  - Suggestion for fix if possible

## Feature Requests
- Include use cases and prioritization
- Link to relevant sprint plan if applicable

## Code Reviews
- All Pull Requests must be reviewed by at least one maintainer
- Fix build failures immediately
- Test changes locally before submitting

## Commit Guidelines
- Write meaningful commit messages (e.g., `chore: fix JWT secret env config`)
- Squash commits if needed
- Avoid redundant merge commits
